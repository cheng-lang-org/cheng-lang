# Cheng 编译提速三段计划

## Summary
- 目标：按截图路线一次性规划 parser per-file 并行、typed expr 全局索引、per-function primary codegen 并行。
- 不新建分支/worktree，不引入 C、脚本、mock、fallback，不跳过当前 `_DirectObjectEmitWriteObject` hard-fail。
- 现有 dirty diff 先保留并复核，不回滚用户/其它助手改动；提速改动要和它们兼容。

## Key Changes
- 只保留 `BACKEND_JOBS` 作为公开并行控制面；新增内部 report 字段：parser/typed/primary 各阶段 `schedule/job_count/task_count/ms`、primary merge 顺序、jobs=1/jobs=N determinism marker。
- Parser：把 source profile 和 expr-layer 构建拆成 per-source task。worker 只读源码并产 `NormalizedExprCallProfile` 或 source expr layer；主线程按 `sourceClosurePaths` 顺序 merge，再串行绑定 import edge target index。
- Typed expr：构建 source context 一次，新增 source/context/range/symbol 索引，替换 `TypedExprSourceContextLookup`、fact/source range、跨文件 return/global 查询里的线性扫描；保留现有 public wrapper，内部走索引。
- Primary codegen：先串行完成 reachability、word count、reloc/data layout；再把每个 reachable function 的 instruction word fill 拆成 task-local `int32[]`，worker 不写共享 `instructionWords`，主线程按函数声明顺序 merge。jobs=1 走同一个 fill 函数作为 oracle。

## Implementation Order
1. 先补基线报告：对 `backend_driver_dispatch_min.cheng` 和普通 fixture 跑 `BACKEND_JOBS=1/4`，记录 parser、typed、primary、merge、RSS、产物 hash。
2. 落 parser 并行：改 `compiler_csg` 的 source loop，失败即 hard-fail；不得 worker 完成顺序影响 expr/fact/function 顺序。
3. 落 typed expr 索引：在 `typed_expr` 内建索引和 source ranges，删除重复 context 重建路径；索引 miss 必须报确定诊断，不能回退全表扫描。
4. 落 primary codegen 并行：抽 `PrimaryFillOneFunctionWords`，接入 work-stealing 或 primary-local worker；worker 只返回 task-local words/error，主线程稳定 merge。
5. 最后同步 `docs/function-parallelism.md`、`task_plan.md/progress.md/findings.md`，只记录已验证事实。

## Test Plan
- 新增/更新 smoke：parser profile 并行 determinism、typed expr index 等价、primary codegen jobs=1/jobs=N hash 等价、worker error hard-fail。
- 必跑：`function_task_contract_smoke`、`function_task_executor_contract_smoke`、`cfg_body_ir_contract_smoke`、`direct_object_writer_smoke`、`perf_memory_contract_smoke`。
- 对同一输入跑 `BACKEND_JOBS=1` 和 `BACKEND_JOBS=4`：比较 `.o/.exe/report` 关键字段和 hash；report 必须显示 `schedule=ws`、`job_count>1`，且不是 standalone no-runtime 假绿。
- 内存验收看 `perf_memory_contract_smoke` 的 RSS、ORC alloc/free/live；任何 worker 泄漏、未 join、错误吞掉都算失败。

## Assumptions
- 目标机器按截图口径默认 8 核 Apple Silicon。
- 第一轮验收以“可证明并行 + 字节确定性 + 阶段耗时下降”为准；500ms-2s 作为后续 perf witness 目标，不在没有新基线前硬写死。
