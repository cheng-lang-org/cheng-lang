# Darwin arm64 函数级并行主线切换

## 摘要
- 目标是在 darwin/arm64 上把 `cheng` 和 `release-compile` 都切到真实的函数级 `ws` 调度，不再只是 `outline + 标签化 ws`。
- 最终行为是一次性切主路径：`BACKEND_FN_SCHED=ws` 成为 dev/release 默认且正式契约；`serial` 只保留给内部诊断、perf 对照和低内存工具链路径，绝不作为 `ws` 的自动回退。
- 核心收益：
  - 把当前“整树语义/mono/ownership + 串行 UIR lower + 串行 isel”拆成函数 task，显著降低大函数拖垮整轮编译的尾延迟。
  - dev 轨可把 `outline` 的价值真正兑现出来，直接服务 `verify_backend_selfhost_100ms_host`。
  - release/system-link 也能复用同一套函数 task 基础设施，在不放松 determinism 的前提下提升吞吐。
  - darwin/arm64 的 direct-exe 路径不再停留在“整块 main object bytes”假并行，而是变成真正的函数块布局与并发回填。

## 接口与契约
- 公共契约更新：
  - `cheng` 默认并固定 `BACKEND_FN_SCHED=ws`。
  - `release-compile` 也默认并固定 `BACKEND_FN_SCHED=ws`。
  - formal spec、tooling README、track env 校验同步改为 `dev=ws`、`release=ws`。
  - `serial` 仅允许内部 gate/诊断入口显式设置，不再作为公开推荐口径。
- 不新增新的公开调度环境变量；继续使用现有 `BACKEND_JOBS` 控制 worker 数。
- 新增内部类型/接口：
  - `UirFnTask`：`func_index`、`decl_ordinal`、`origin_file`、`body_tok_start/end`、`cost_hint`、`generic_epoch`。
  - `UirFnTaskResult`：`func_index`、`lowered_func`、`cstrings`、`obj_alias_updates`、`generic_requests`、`diag_bundle`、`profile_counters`。
  - `UirCoreSharedSnapshot`：冻结后的函数/全局/类型/对象布局/overload/entry-path 查找表，供 worker 只读访问。
  - `DirectExePlan`：有序 `function_chunks`、`reloc_patches`、`cstrings`、`globals`、`imports`，替代当前 `machoDirectWriteHostExe(mainObjBytes, ...)` 的占位式输入。

## 实现变更
- Stage1/语义拆分：
  - 保留 `outline` 解析，但取消“先整模块 materialize bodies 再整树 semantics/mono/ownership”的路径。
  - 把 `checkSemanticsId` 拆成两段：`collect_globals + bind_imports + policy` 串行；`routine body analyze` 改成函数级 API，worker 只处理单个例程。
  - 把 `monomorphizeId` 改成固定点任务模型：worker 在分析/降级时只产出 `GenericInstanceKey` 请求；主线程按稳定 key 顺序去重、建新 task、推进到 fixed-point。
  - 把 `ownershipAnalyzeId` 改成 `ownershipAnalyzeRoutineBody`，结果只回写到当前函数的 proof/diag，不再依赖整树全局可变数组。
  - worker 不允许修改共享 AST SoA。对 deferred body 采用“按 token span 解析到 worker-local AST 片段”的方式，避免对共享 `AstNodeId` 做写操作。
- UIR builder：
  - 保留现有串行 `predeclare_top_level`，把它变成函数任务的唯一 header/source-of-truth。
  - `lower_top_level` 改成两段：非函数 top-level 串行消费；函数 body lowering 走 `UirFnTask` work-stealing。
  - `uirCoreBuildFunc` 改成纯任务执行接口，不再直接写共享 module/global symbol state；所有 cstring/alias/type-key 增量改为 task-local产出，主线程按 `func_index/decl_ordinal` 串行 merge。
  - `fn_tasks` profile 真正记录 enqueue/run/wait/merge，而不是空 checkpoint。
- 调度器：
  - 新增 darwin/arm64 host-only 原生 work-stealing bridge，使用 `pthread + per-worker deque + steal`。
  - 初始分发按 `body_tok_span` 降序做 seed，worker 本地 LIFO、跨 worker FIFO steal。
  - `ws` 一旦命中 worker panic/bridge error/merge 冲突，立即 hard-fail；不做 serial fallback。
- UIR opt / codegen：
  - 把纯 per-function 的 UIR pass 统一经 `for-each-func ws` 执行；跨函数决策与汇总保持串行。
  - `aarch64LowerFunc`、future regalloc、machine func 汇总改成“并行产出 `MachineFunc`，串行按原声明顺序 merge”。
  - writer 侧继续保持符号表、section 顺序、cstring/global 排布的确定性；执行顺序不可影响最终 `.o` / `.exe` 字节流。
- Direct-exe：
  - 把 `macho_direct_exe_writer` 从 facade 改成真实实现：串行计算 `DirectExePlan` 布局，worker 并发做函数 text/reloc patch 回填，最终 Mach-O header/load command/symtab 串行组装。
  - dev self-link/direct-exe 直接吃函数块计划；release/system-link 继续走 object-first，但 object 由同一套函数 task codegen 生成。
- Tooling/文档：
  - 更新 `cheng_tooling` 的 fixed-track env 校验、`build-backend-driver` 约束、`backend_prod_closure` 说明和相关 docs。
  - 更新 profile schema/baseline 生成器，纳入新的 `fn_tasks / isel_tasks / direct_exe_*` marker。
  - 更新 `docs/cheng-plan-full.md` 和 `docs/UIR.md`，把“函数级并发”从目标态改成已实现架构说明。

## 测试与验收
- 功能正确性：
  - 多文件、多函数、重载、`importc`、对象返回、global-init、generic helper、trait/template、fixups 全量通过。
  - 新增 worker 失败夹具，断言 `ws` 下直接失败且不会静默回退串行。
  - dev direct-exe 与 release/system-link 都要编译、运行、通过现有闭环 fixture。
- 确定性：
  - `verify_backend_determinism_strict` / `verify_backend_exe_determinism_strict` 增加 `BACKEND_JOBS=1` 与默认 jobs 对照，要求 SHA 一致。
  - dev self-link/direct-exe 也增加 `TMPDIR/TZ/jobs` 扰动下的重复构建一致性检查。
  - `soa_report`、SIMD、egraph、UIR stability 等双跑 gate 保持无漂移。
- 性能：
  - 主 KPI 是 `verify_backend_selfhost_100ms_host`；以该 gate 和 compile-stage1 冷态耗时作为调优主目标。
  - `verify_backend_selfhost_parallel_perf` 改成新 `ws` 对显式 `serial` 的对照 gate，用来保证并行不比串行更差。
  - `multi_perf_regression` 与 profile baseline 只在专用 perf 机器上刷新；刷新条件是 `ws` 路径稳定且 `100ms` 主 KPI 明显改善。

## 假设与默认
- 首个一等公民平台是 darwin/arm64；其他平台暂不要求同步拿到函数级 direct-exe。
- 这是一次性切主路径，不接受“部分子系统仍串行但对外宣称函数级并行”。
- release 轨一起并行化，并同步修改 formal spec、tooling contract 和 determinism gate。
- `serial` 只保留给内部 bring-up、perf 对照和低内存约束场景；`ws` 失败时绝不自动降级。
