## codegen bug 发现与 workaround

在 `StatementScanChecksum` while 循环内访问 `scan.exprTokenCount` 会破坏
另一个函数 `StatementExprCheck` 的执行（exit 181 vs 42）。

ARM64 指令完全相同，BL patch 解析已验证正确，根因未找到
（怀疑 macOS page-level code signing 效应）。

**Workaround**：将问题字段访问隔离到独立函数 `StatementScanChecksumExprPart`。
`StatementScanChecksum` 调用该函数获取 `exprTokenCount` 累加值，
回避主循环内直接访问。
