## What was done
Added import alias fallback to the cold compiler so that when a bare name (e.g., `Assert`) is not found in the symbol table, the parser iterates the module's import aliases and tries `alias.name` (e.g., `system.Assert`).

### Changes in `bootstrap/cheng_cold.c`
1. **Forward-declared `struct ColdImportSource`** before the `Parser` struct
2. **Added fields** `import_sources` and `import_source_count` to `Parser` struct
3. **Moved `ColdImportSource` definition** to right after `Parser` struct (was ~line 19942)
4. **Populated `parser.import_sources`** in:
   - `cold_compile_one_import_direct` (import-mode parser, collects sub-imports from the module being compiled)
   - Entry module parser in `cold_compile_source_path_to_macho`
   - Entry module parser in `cold_compile_source_to_object`
5. **Added import alias fallback** in `parse_call_after_name` and `parse_call_from_args_span`:
   - When `fn_index < 0`, if `parser->import_source_count > 0`, iterate over `parser->import_sources`
   - For each import source, construct `alias.name` and try `symbols_find_fn_for_call`
   - If found, use the resolved `fn_index`
