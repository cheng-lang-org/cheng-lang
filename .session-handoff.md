## Known cold compiler bug: cross-function codegen corruption

### Symptom
Adding `scan.exprTokenCount` to the `lineScore` sum in `StatementScanChecksum` causes `StatementExprCheck` to fail (exit 181 instead of 42).

### Reproduction
- Working: `let lineScore = kindScore + scan.indent + scan.payloadLen`
- Broken: `let lineScore = kindScore + scan.indent + scan.payloadLen + scan.exprTokenCount`
- Just ONE extra field access in an addition triggers the bug
- The bug affects a DIFFERENT function (StatementExprCheck), not the modified one
- Both functions use `StatementScan` struct
- `StatementScanChecksum` is called from `StatementCheck` only
- `StatementExprCheck` is called from `main()` independently

### Observations
- 28/28 regression still passes
- Binary disassembly has identical ARM64 mnemonics (only extra ops for the added field)
- No errors/warnings during compilation
- `cold_frontend_function_count=136` same in both versions
- Bug is deterministic (COLD_NO_SIGN=1 produces bit-identical output)

### Hypotheses
1. Composite call ABI (str passing) stack frame corruption
2. Slot allocation overflow when adding field accesses to a function
3. Arena/buffer overflow in BodyIR
4. Register allocation conflict

### Workaround
Use range check instead of exact checksum: `if observed < 10000 || observed > 20000`
